# 普通模式(默认模式) / 权限测试模式

## 普通模式

普通模式下,功能和原有功能一致,业务逻辑不变


## 权限测试模式

权限测试模式下,用户可以配置多个用户会话信息,用户从其他位置发送到插件的报文会遍历这些用户会话信息,将这些会话信息替换掉原始的请求中的会话信息,然后发送

### 用户会话信息

用户会话信息可以是header中的字段,也可以是json中的某些个字段或者是xml中的某些个字段,或者请求表单中的某些个字段,有可能是headers中
和body中同时存在用户会话信息

### 判决规则

用户还可以编辑请求成功与否(即越权成功与否的判断)的规则,规则可以对响应码进行判断或者是响应头或者响应体进行提取判断,判断规则包含成功的情况下标记颜色和备注信息和失败的情况下的标记颜色和备注信息.默认成功标记红色不成功绿色.默认备注信息不填写.
默认没有判断规则的情况下使用响应报文的响应码和响应报文的相似度判决是否存在越权问题或者说是否成功越权了

规则管理还能搜索过滤(简单的关键词匹配和高级搜索匹配),还要能开启或关闭规则,还能导入导出规则.导入导出规则文件是yaml格式的文件

### 重放配置 
权限测试模式下,用户可以控制用户一从其他地方发送过来里面遍历用户会话信息重放请求也能等都发送过来完成了再一起重放
用户还能配置勾选去重功能,重复的接口再发过来就不管了

### 自动化 scope
权限测试模式下用户还能管理维护一个scope(用户自定义的scope)去自动的从history拿到报文自动遍历用户会话重放请求进行测试,也能勾选直接使用Burpsuite本身的scope.

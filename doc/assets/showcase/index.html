原生 Repeater 的痛点

测试效率
被无形消耗

🔴 数据易丢失

关闭 Burp Suite 后所有请求和历史记录全部消失，无法持久化保存，每次重启从零开始

🔴 无历史对比

无法自动记录每次重放的响应变化，缺少差异对比工具，难以追踪测试演进过程

🔴 管理混乱

大量请求堆积在标签页中，缺乏分类、搜索、标记和批量操作能力

🔴 越权测试繁琐

手动替换 Token 进行水平/垂直越权测试，效率低下且极容易遗漏

# 《取经人·重放管理传》—— Repeater Manager 宣传剧本

> 以西游记故事框架演绎 Repeater Manager 的诞生与使命

---

## 第一回：混沌初开·乱象丛生

**【场景：东土大唐·网络安全界】**

话说东土大唐，网络安全测试之士众矣。然众修士每日面对 Burp Suite 原生 Repeater，苦不堪言——

> 请求散乱如野猴，无章无法，寻一旧报文如大海捞针；
> 重放之后无记忆，昨日之响应今日无处可寻；
> 数据如流沙，Burp 一关，辛苦整理尽归虚无；
> 越权测试如盲人摸象，换 Token、比响应，全凭手工，耗时耗力。

众修士哀叹曰："吾等每日与 HTTP 报文搏斗，如徒手攀天梯，何时得正果？"

**【画外音】**

> 此乃测试人员之"八十一难"第一难——**管理之乱**。请求无序、历史无存、数据易失，修士空有一身本领，却困于工具之弊。

| 数据持久化 | 定海神针 | SQLite 本地存储，Burp 关了依然在 |

众修士见之，纷纷侧目："此乃何方神圣？竟能解我多年之苦！"

**【画外音】**

> 初出茅庐，便以三大绝技震惊四座——**管理有序、历史可查、数据永存**。此乃"齐天大圣"之雏形也。

## 第四回：大闹天宫·越权测试

**【场景：天宫·越权测试战场】**

Repeater Manager 艺成下山，直闯天宫——此乃安全测试之核心战场：**越权测试**。

天宫之中，等级森严：

- **普通天兵**（低权限用户）只能巡视南天门
- **天将**（中权限用户）可调遣部分兵马
- **玉帝**（高权限用户）统御三界

然天宫之门户众多，常有门户把关不严，低权限者误入高权限之地——此谓**越权漏洞**也。

Repeater Manager 携其"越权测试军团"大闹天宫：

### 先锋大将：SessionManager（多用户会话管理）

> 如孙悟空拔毫毛变出无数小猴，配置多组用户凭证，各带 Token，整装待发。

### 军师：TokenReplacementEngine（Token 替换引擎）

> 如孙悟空七十二变，将请求中原有 Token 替换为其他用户之 Token，神不知鬼不觉。

- 支持 Header / Cookie / Body / URL 参数 四种位置
- 精准定位，一键替换

### 判官：JudgmentEngine（响应判断引擎）

> 如阎罗王判生死，根据状态码、响应体、响应头、响应时间判断越权是否成功。

- CONTAINS / NOT_CONTAINS / REGEX / LENGTH_DIFF 四法并用
- 智能比对，自动标记风险等级

### 元帅：AutoTestEngine（自动化测试引擎）

> 如孙悟空一个筋斗云十万八千里，拦截代理流量 → 自动替换 Token → 重放请求 → 判断结果，全流程自动化。

**【战斗场面】**

Repeater Manager 坐镇中军，ScopeManager 划定战场，AutoTestEngine 率众将冲锋陷阵——

1. **探**：拦截符合范围之请求
2. **变**：TokenReplacementEngine 将天兵之 Token 换为天将之 Token
3. **攻**：ReplayEngine 重放请求至天宫服务器
4. **判**：JudgmentEngine 比对响应，若天兵得天将之响应，越权漏洞现形！
5. **标**：结果以颜色标记，风险等级一目了然

**【画外音】**

> 大闹天宫一战，以自动化之力，将昔日需数日之越权测试压缩至数分钟。水平越权、垂直越权，无所遁形。此乃"斗战胜佛"之威名由来。

## 第五回：五行山下·内功大成

**【场景：五行山·工程化修炼】**

大闹天宫之后，Repeater Manager 归于五行山下——非为惩罚，乃为沉淀打磨，修炼内功。

### 五大内功

| 内功 | 对应服务 | 妙用 |
|------|----------|------|
| 不动如山 | AutoSaveService | 定时将内存数据同步至磁盘，防意外丢失 |
| 净化之术 | GarbageCollectorService | 后台清理 Pool 零引用数据，十日一循环 |
| 金蝉脱壳 | HistoryRecordingService | 异步队列写入历史，不阻塞主线程 |
| 七十二洞 | 数据库连接池 | BlockingQueue + JDK Proxy，连接复用 |
| 历劫不灭 | Schema 版本化迁移 | 升级无忧，数据不丢 |

**【画外音】**

> 五行山下五百载，内功大成。自动保存防丢、垃圾回收省空间、异步队列保流畅——此乃工程化之真谛。

## 第六回：西天取经·七部真经

**【场景：西天雷音寺·正式发布】**

历经九九八十一难，Repeater Manager 终成正果，于雷音寺受封——

**"Burp Suite 请求重放管理斗战胜佛"**

其携七部真经（JAR 包）返回东土，普度众生：

| 真经 | 妙用 | 譬喻 |
|------|------|------|
| API 安全测试 | 持续测试同一 API 不同参数组合，历史比对一目了然 | 反复诵经，字字珠玑 |
| API 路径整理 | 自动从不规则请求中提取标准 API 路径 | 取经路上整理经文 |
| 越权漏洞检测 | 配置多用户会话，自动检测水平/垂直越权 | 火眼金睛辨妖魔 |
| 漏洞复现 | 记录漏洞利用全过程，便于后期复现 | 唐僧取经笔记 |
| 安全评估 | 整理大型应用 API 集合，系统化测试 | 取经团队分工协作 |
| 团队协作 | ERM 存档 + 全局 YAML 规则共享 | 经文传抄流通 |
| 渗透测试记录 | 记录关键请求，便于编写报告 | 取经归来著书立说 |

**【安装指引】**

欲得此真经者，只需——

1. 从 Releases 页面下载 JAR 文件
2. 打开 Burp Suite Professional
3. Extensions → Installed → Add → 选择 JAR
4. Next → 功德圆满

**【画外音】**

> 取经归来，七部真经造福网络安全界。无论你是初出茅庐之安全学徒，还是身经百战之渗透大师，此插件皆能助你一臂之力。

## 第七回：旃檀功德·未来可期

**【场景：大雷音寺·未来展望】**

如来佛祖拈花微笑，曰："Repeater Manager 虽已成正果，然取经之路未有穷期。"

未来之蓝图已然绘就：

- **请求序列功能**：支持多步骤请求流程，如取经团队一路西行，一站接一站
- **请求模板功能**：快速创建类似请求，如孙悟空拔毫毛变分身
- **更多数据格式导入导出**：与更多工具互通有无
- **团队规则共享云端同步**：真经不再局限于一人之手，云端共享，全球同步
- **集成外部漏洞扫描器结果对比**：与各路神仙联手，共降妖除魔

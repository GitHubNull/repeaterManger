# 《取经人·重放管理传》—— Repeater Manager 宣传剧本

> 以西游记故事框架演绎 Repeater Manager 的诞生与使命

---

## 第一回：混沌初开·乱象丛生

**【场景：东土大唐·网络安全界】**

话说东土大唐，网络安全测试之士众矣。然众修士每日面对 Burp Suite 原生 Repeater，苦不堪言——

> 请求散乱如野猴，无章无法，寻一旧报文如大海捞针；
> 重放之后无记忆，昨日之响应今日无处可寻；
> 数据如流沙，Burp 一关，辛苦整理尽归虚无；
> 越权测试如盲人摸象，换 Token、比响应，全凭手工，耗时耗力；
> 两份响应欲细较，肉眼逐行，头昏眼花；
> 测罢漏洞欲成册，手动排版，劳心费神。

众修士哀叹曰："吾等每日与 HTTP 报文搏斗，如徒手攀天梯，何时得正果？"

**【画外音】**

> 此乃测试人员之"八十一难"第一难——**管理之乱**。请求无序、历史无存、数据易失、对比无器、报告难成，修士空有一身本领，却困于工具之弊。

**【场景：花果山·项目立项】**

忽有一日，花果山顶仙石崩裂，一道金光直冲云霄——

**Repeater Manager** 应运而生！

此插件秉承 Montoya SDK 之正统，集 Java 17 之灵气，携 SQLite 之坚韧，自 Burp Suite 扩展之土壤中破石而出。

其出世之日，天地为之震动：

| 绝技 | 神通 | 妙用 |
|------|------|------|
| 请求管理 | 七十二变 | 分类、标记、备注，五色标签一念之间；多选批量操作，删改自如 |
| 历史记录 | 紧箍咒 | 每次重放之响应皆自动存档，永不遗忘；多条件筛选，瞬息定位 |
| 数据持久化 | 定海神针 | SQLite 本地存储，Burp 关了依然在；Pool 去重，省空间如泰山 |
| 消息比对 | 照妖镜 | 两份报文并排放置，行级差异、字符级差异一目了然 |
| 报告生成 | 天书玉册 | 越权测试结果一键导出 PDF/HTML/Markdown，附 cURL 与 Postman 代码 |

众修士见之，纷纷侧目："此乃何方神圣？竟能解我多年之苦！"

**【画外音】**

> 初出茅庐，便以五大绝技震惊四座——**管理有序、历史可查、数据永存、比对有器、报告自成**。此乃"齐天大圣"之雏形也。

## 第三回：灵台问道·习得神通

**【场景：灵台方寸山·功能进阶】**

Repeater Manager 不满足于初生之能，遂拜入菩提祖师门下。祖师见其根骨不凡，遂授以六大神通——

### 神通一：筋斗云·Pool 去重架构

> 一个筋斗十万八千里，重复内容只存一份

string_pool、header_pool、body_pool、file_pool 四层去重，SHA-256 哈希为引，引用计数为绳。昔日存储空间如流水，今日去重之后，磁盘稳如泰山。更有垃圾回收之术，零引用数据十日一清，自动净化。

### 神通二：火眼金睛·高级搜索

> 炼就金睛，妖魔鬼怪无处遁形

请求列表实时关键词/正则搜索，支持大小写切换；历史记录多条件复合筛选——方法、协议、域名、路径、状态码范围、响应长度范围、备注、越权标记，诸条件 AND 组合，一念即达。日志面板亦带搜索导航，五级日志一目了然。

### 神通三：分身术·API 规则提取

> 拔一根毫毛，吹出猴万个

四提取源（URL_PATH、URL_QUERY、HEADER、BODY）× 四提取法（REGEX、SUBSTR、JSON_PATH、XPATH），规则引擎自动从不规则请求中提取标准 API 路径。项目级规则存于 SQLite，全局规则存于 YAML，内存临时规则三者合一，三十秒缓存，去重合并，如分身遍查三界。

### 神通四：法天象地·布局切换

> 身长万丈，小如微尘，随心而变

左右并列、上下排列、仅显请求、仅显响应——四种形态一键切换，适应不同屏幕与场景。列显示控制更可自由显隐非必要字段，信息密度尽在掌握。

### 神通五：乾坤袋·数据导入导出

> 内有乾坤，纳须弥于芥子

ERM 加密存档如紫金葫芦——自定义二进制格式，Deflate 压缩，可选 AES-256-CBC + HMAC-SHA256 加密（PBKDF2 密钥派生），数据库与 blob 文件一包带走。Postman Collection v2.1 互通有无，智能导入自动识别格式。

### 神通六：千里眼顺风耳·日志与代理

> 多通道输出，五级分明，洞察秋毫；配置 HTTP 代理，调试畅通无阻

Burp 控制台、滚动文件、UI 面板三通道独立开关，DEBUG/INFO/SUCCESS/WARN/ERROR 五级过滤。UI 日志终端风格，暗色背景，彩色分级，支持搜索导航、自动滚动、暂停导出。HTTP 代理调试，SSL 全信任，请求收发尽在掌控。

**【画外音】**

> 六大神通、四般技艺学成，Repeater Manager 已非吴下阿蒙。众修士用之，如虎添翼，效率倍增。

## 第四回：大闹天宫·越权测试

**【场景：天宫·越权测试战场】**

Repeater Manager 艺成下山，直闯天宫——此乃安全测试之核心战场：**越权测试**。

天宫之中，等级森严：

- **普通天兵**（低权限用户）只能巡视南天门
- **天将**（中权限用户）可调遣部分兵马
- **玉帝**（高权限用户）统御三界

然天宫之门户众多，常有门户把关不严，低权限者误入高权限之地——此谓**越权漏洞**也。

Repeater Manager 携其"越权测试军团"大闹天宫：

### 先锋大将：SessionManager（多用户会话管理）

> 如孙悟空拔毫毛变出无数小猴，配置多组用户凭证，各带 Token，整装待发。

支持会话颜色标记、启用/禁用开关；Token 位置六处可设——Header、JSON 正文、XML 正文、表单字段、多部分字段、URL 参数；全局 Token 位置 YAML 共享，跨会话复用。

### 军师：TokenReplacementEngine（Token 替换引擎）

> 如孙悟空七十二变，将请求中原有 Token 替换为其他用户之 Token，神不知鬼不觉。

六处位置精准定位，类型感知解析；相似度阈值可调（默认 0.7），API 去重防重复处理；实时模式与批量模式两相宜。

### 判官：JudgmentEngine（响应判断引擎）

> 如阎罗王判生死，根据状态码、响应体、响应头、响应时间判断越权是否成功。

九种判法并用：CONTAINS、NOT_CONTAINS、REGEX、EQUALS、NOT_EQUALS、GREATER_THAN、LESS_THAN、NUMERIC_EQUALS、LENGTH_DIFF。优先级执行，首中即决；默认兜底——状态码比对 + Levenshtein 相似度计算。每规则可自定义成败颜色与备注。

### 元帅：AutoTestEngine（自动化测试引擎）

> 如孙悟空一个筋斗云十万八千里，拦截代理流量 → 自动替换 Token → 重放请求 → 判断结果，全流程自动化。

ScopeManager 划定战场，URL 模式匹配；代理流量实时拦截，符合范围即触发；多会话顺序替换，批量执行不阻塞。

### 后勤：批量操作

> 如天兵天将齐上阵，批量重放、批量越权测试、批量删除——多选历史记录或请求，一键执行，进度实时显示。

**【战斗场面】**

Repeater Manager 坐镇中军，ScopeManager 划定战场，AutoTestEngine 率众将冲锋陷阵——

1. **探**：拦截符合范围之代理请求
2. **变**：TokenReplacementEngine 将天兵之 Token 换为天将之 Token（六处位置无一遗漏）
3. **攻**：ReplayEngine 重放请求至天宫服务器
4. **判**：JudgmentEngine 以九种判法比对响应，若天兵得天将之响应，越权漏洞现形！
5. **标**：结果以颜色标记，风险等级一目了然；状态栏实时显示模式、耗时、批量进度
6. **比**：可疑结果送入 Diff 引擎，原始响应与替换响应并排放置，差异无处藏身
7. **报**：一键生成 PDF/HTML/Markdown 报告，附 cURL 命令与 Postman 代码片段，交付即成

**【画外音】**

> 大闹天宫一战，以自动化之力，将昔日需数日之越权测试压缩至数分钟。水平越权、垂直越权，无所遁形。此乃"斗战胜佛"之威名由来。

## 第五回：照妖镜前·Diff 显形

**【场景：盘丝洞·消息比对战场】**

话说取经路上，妖魔鬼怪善变化，真假难辨。Repeater Manager 取出**照妖镜**——Diff 引擎，令一切伪装无所遁形。

### 行级比对·LCS 算法

> 最长公共子序列，增删改一目了然

两份报文左右并置，绿色为未改、红色为删除、浅绿为新增、黄色为变更。变更之行更以字符级高亮，匹配处翠绿、差异处朱红（原稿）/墨绿（修改稿），字字分明。

### 字节级比对·Hex 模式

> 妖魔幻化二进制，Hex _dump 现原形

偏移量、十六进制、ASCII 三栏并列，字节级差异精准定位。非文本响应亦可比对。

### 四 pane 模式·全景对比

> 双 tab + 四格，请求响应同时较

请求对比 tab、响应对比 tab 自由切换；四 pane 模式 2×2 网格——原始请求、替换请求、原始响应、替换响应同时呈现，全局在握。

### 同步滚动·搜索导航

> 左右同步如双马并驰，差异跳转如按图索骥

SynchronizedScrollPanel 左右联动，DiffNavigator 上一处/下一处变更，SearchBar Ctrl+F 关键词/正则搜索，匹配数实时显示。全屏放大，90% 屏幕尽享细节；相似度百分比，量化差异程度。

**【画外音】**

> 照妖镜前，无论妖魔幻化何种形态，行级差异、字符差异、字节差异层层剥开，真相大白。此乃测试人员之"火眼金睛"也。

## 第六回：天书玉册·报告自成

**【场景：雷音寺藏经阁·报告生成】**

越权测试毕，需著书立说，呈报天庭。Repeater Manager 取出**天书玉册**——报告生成系统，一键成册。

### 三般格式

| 格式 | 神通 | 妙用 |
|------|------|------|
| PDF | 金刚不坏 | Apache PDFBox 3.0.1，自动探测系统 CJK 中文字体，中文报告不乱码 |
| HTML | 千变万化 | FreeMarker 模板，CSS 自包含，浏览器即开即阅 |
| Markdown | 大道至简 | FreeMarker 模板，简洁明了，版本控制友好 |

### 册中内容

- **接口分组**：按 API 端点聚合结果，条理清晰
- **基线数据**：原始请求/响应完整收录，对比有凭
- **会话发现**：每用户会话之结果、判定、相似度、状态码一一罗列
- **cURL 命令**：自动生成，复现漏洞一键复制
- **Postman 代码**：导入即用 JSON 片段
- **统计摘要**：总测试数、越权数、安全数、错误数、基线数、独立端点数，尽收眼底
- **会话分解**：每会话计数汇总

### 导出三模式

- **明文**：原始文件直出
- **压缩**：.ermr 格式，体积小巧
- **加密压缩**：.ermr + 密码，AES-256 守护，敏感数据不外泄

导出前敏感数据警示，导出时进度对话框，报告亦可解密回读。

**【画外音】**

> 昔日测试完毕，报告需手动整理数日；今日天书玉册，一键生成三般格式，附复现命令与代码片段。交付之速，如神笔马良。

## 第七回：五行山下·内功大成

**【场景：五行山·工程化修炼】**

大闹天宫之后，Repeater Manager 归于五行山下——非为惩罚，乃为沉淀打磨，修炼内功。

### 六大内功

| 内功 | 对应服务 | 妙用 |
|------|----------|------|
| 不动如山 | AutoSaveService | 定时数据库 checkpoint（默认 5 分钟），防意外丢失 |
| 净化之术 | GarbageCollectorService | 后台清理 Pool 零引用数据，十日一循环，批量处理 |
| 金蝉脱壳 | HistoryRecordingService | 异步队列写入历史，单后台线程，不阻塞主线程 |
| 七十二洞 | 数据库连接池 | BlockingQueue + JDK 动态代理，5 连接复用，透明归还 |
| 历劫不灭 | Schema 版本化迁移 | 升级无忧，数据不丢，schema_meta 记录版本与清洁关机 |
| 遁地术 | 代理调试 | HTTP 代理配置，SSL 全信任，调试请求收发 |

### 额外护体神功

- **状态面板**：模式指示（正常/越权测试彩色标识）、请求状态（对勾/叉号）、响应大小、时间戳、耗时毫秒、批量进度
- **使用教程面板**：CommonMark Markdown 渲染，GFM 表格支持，中英双语切换，快速入门与详细教程两档
- **关于面板**：版本信息、作者许可、GitHub 链接、依赖清单
- **右键菜单**：Burp 上下文菜单"Send to Repeater Manager"、"Send to Privilege Test"，批量请求支持

**【画外音】**

> 五行山下五百载，内功大成。自动保存防丢、垃圾回收省空间、异步队列保流畅、连接池复用高效、Schema 迁移保数据——此乃工程化之真谛。

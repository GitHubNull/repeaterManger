# 快速入门教程

> 5 分钟上手 Repeater Manager

## 1. 安装插件

1. 从 [Releases](../../releases) 下载最新版 JAR 文件
2. 打开 Burp Suite Professional
3. 进入 `Extensions` → `Installed` → 点击 `Add`
4. 选择下载的 JAR 文件 → 点击 `Next`
5. 看到 **"Repeater Manager 插件加载成功"** 即安装完成

> 首次加载会自动在 `~/.burp/` 下创建会话目录，包含数据库文件和日志。

## 2. 发送请求

在 Burp Suite 任意位置，右键点击 HTTP 请求：

- 选择 **"发送到 Repeater Manager"**
- 切换到顶部 **"Repeater Manager"** 标签页

## 3. 编辑和重放

- 右上区域：编辑请求内容（支持语法高亮）
- 点击 **"发送"** 按钮重放请求
- 右侧面板显示响应内容
- 底部状态栏显示响应耗时和大小

## 4. 查看历史

- 左下方面板：查看该请求的所有历史重放记录
- 双击历史记录项：加载该次重放的请求和响应
- 可使用高级搜索（多条件筛选）快速定位
- 右键菜单支持列控制、颜色标记等操作

## 5. 管理请求

- **颜色标记**：右键请求列表中的条目，选择颜色
- **备注**：为请求添加注释说明
- **搜索**：在搜索框中输入关键词快速定位
- **新建请求**：点击顶部工具栏按钮
- **列控制**：自定义表格显示列

## 6. API 规则提取

- 进入 **"配置"** → **"API 规则配置"** 标签页
- 添加全局规则（跨会话共享，YAML 存储）或项目规则（SQLite 存储）
- 配置提取源（URL路径/查询参数/头部/Body）和提取方法（正则/子串/JSONPath/XPath）
- 通过右键菜单触发提取，或规则变更后自动重提取
- 提取结果可在请求列表中查看和管理

## 7. 越权测试

- 进入 **"配置"** 面板，配置：
  - **用户会话**：添加不同权限用户（凭证/Token 及 Token 位置）
  - **判断规则**：设置越权成功判断条件（状态码/响应体/响应头/响应时间）
  - **请求范围**：指定需要测试的 URL 模式
- 开启自动检测，插件会拦截匹配范围的代理流量
- 自动替换 Token 后重放请求，根据规则判断越权风险
- 在越权测试面板查看结果（颜色标记：红=疑似越权、绿=安全）

## 8. 导出数据

进入 **"配置"** → **"数据导入导出"**：

- **ERM 存档**：导出/导入插件专用格式，支持 AES-256 加密
- **Postman**：导出为 Postman Collection v2.1 格式
- **智能导入**：自动检测文件格式

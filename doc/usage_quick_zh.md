# 快速入门教程

> 5 分钟上手 Repeater Manager

---

## 1. 安装插件

1. 从 [Releases](../../releases) 下载最新版 JAR 文件
2. 打开 Burp Suite Professional
3. 进入 `Extensions` → `Installed` → 点击 `Add`
4. 选择下载的 JAR 文件 → 点击 `Next`
5. 看到 **"Repeater Manager 插件加载成功"** 即安装完成

> 首次加载会自动在 `~/.burp/` 下创建会话目录，包含数据库文件和日志。

---

## 2. 发送请求到插件

在 Burp Suite 任意位置（Proxy/Intruder/Scanner 等），右键点击 HTTP 请求：

- 选择 **"发送到 Repeater Manager"**
- 切换到顶部 **"Repeater Manager"** 标签页，即可看到请求已添加到列表

---

## 3. 编辑与重放请求

- **右侧上方**：编辑请求内容（支持语法高亮）
- 点击 **"发送"** 按钮重放请求
- **右侧下方/右侧**：显示响应内容
- **底部状态栏**：显示响应耗时和大小

---

## 4. 查看历史记录

- **左下方面板**：查看该请求的所有历史重放记录
- 记录按时间倒序排列，显示状态码、响应长度、响应时间
- **双击** 历史记录项：加载该次重放的请求和响应
- 使用 **高级搜索**（多条件筛选）快速定位记录

---

## 5. 管理请求

- **颜色标记**：右键请求列表中的条目，选择颜色进行分类
- **备注**：为请求添加注释说明
- **搜索**：在搜索框中输入关键词快速定位请求
- **新建请求**：点击顶部工具栏的 **"新建请求"** 按钮
- **删除请求**：右键选择删除，清理不需要的请求

---

## 6. 切换布局

点击右上角 **"布局"** 下拉菜单，选择适合的显示方式：

| 布局 | 说明 |
|------|------|
| 左右布局 | 请求在左，响应在右（默认） |
| 上下布局 | 请求在上，响应在下 |
| 仅请求 | 只显示请求编辑区 |
| 仅响应 | 只显示响应展示区 |

---

## 7. 列显示控制

- 右键请求列表的表头，打开 **"列控制"** 对话框
- 勾选/取消勾选以显示或隐藏特定列
- 常用列包括：ID、方法、域名、路径、状态码、响应长度、颜色、备注等

---

## 8. 报文比对

- 在历史记录面板右键选择 **"报文比对"**，对比原始请求/响应与越权测试报文
- 支持 **标签页模式**（请求/响应分标签）和 **四分格模式**（原始/替换请求和响应同时展示）
- 差异高亮显示（绿色新增、红色删除、黄色修改），支持字符级行内差异
- **同步滚动**：原始和替换两端面板自动联动
- **差异导航**：使用上/下一处差异按钮快速跳转
- **搜索**：使用可折叠搜索栏在差异内容中搜索关键词，支持正则表达式

---

## 9. 批量操作

- 在历史记录面板按住 **Ctrl/Shift** 多选条目
- 右键选择 **"批量重放"** 依次重新发送所选请求
- 右键选择 **"批量越权测试"** 自动遍历所有用户会话进行检测
- 右键选择 **"删除"** 批量清理不需要的记录

---

## 10. 报告导出

- 在越权测试面板点击 **"导出报告"**，将测试结果导出为正式报告
- 支持四种格式：
  | 格式 | 特点 |
  |------|------|
  | PDF | 原生格式，内嵌中文字体，适合正式交付 |
  | HTML | 浏览器直接打开，样式美观 |
  | Markdown | 纯文本，适合版本控制和二次处理 |
  | ERMR | 加密容器，AES-256-CBC 加密，适合安全传输 |
- 报告内容包括：测试概要、会话统计、逐端点详述、cURL/Postman 代码片段
- 判决结果显示为中文名称：⚠ 越权 / ✔ 安全 / ✗ 错误，命中规则组名称同时展示
- 请求/响应体自动渲染（文本高亮、二进制 hex 转换）

---

## 11. 配置存储与日志

进入 **"配置"** 标签页：

### 存储配置
- **存储模式**：自动（默认）/ 指定目录 / 指定文件
- **自动保存**：默认每 5 分钟自动同步数据到磁盘

### 日志配置
- **日志级别**：DEBUG / INFO / SUCCESS / WARN / ERROR
- **日志通道**：Burp 控制台 / 文件日志 / UI 面板日志
- 日志文件保存在会话目录的 `logs/` 子目录下

---

## 12. 导入导出数据

进入 **"配置"** → **"数据导入导出"** 标签页：

- **ERM 存档**：插件专用格式，支持 AES-256 加密，适合备份和团队共享
- **Postman Collection**：导出为 v2.1 格式，方便与其他工具协作
- **智能导入**：自动检测文件格式（.erm / .json）

> 更新插件前建议先导出 ERM 存档备份数据。

---

## 13. API 规则提取

插件可从非标准请求中自动提取 API 路径，便于整理和管理：

1. 进入 **"配置"** → **"API 规则配置"** 标签页
2. 添加 **全局规则**（跨会话共享，YAML 存储）或 **项目规则**（当前会话独立存储）
3. 配置提取源（URL路径/查询参数/头部/Body）和提取方法（正则/子串/JSONPath/XPath）
4. 通过右键菜单触发提取，或规则变更后自动重提取
5. 提取结果可在请求列表中查看和管理

---

## 14. 去重配置

配置去重策略避免同一 API 被重复越权测试：

- 进入 **"配置"** → **"越权测试"** → **"去重配置"** 标签页添加去重规则
- 支持 6 种去重策略：PATH（URL 路径）/ API（提取路径）/ JSON_BODY_FIELD / XML_BODY_FIELD / FORM_FIELD / URL_PARAM
- 3 种保留策略：FIRST（保留首个）/ LAST（保留末个）/ MIDDLE（保留中间）
- 按优先级链式匹配，首个匹配到的规则生效

---

## 15. 越权测试

自动化越权漏洞检测功能：

1. 进入 **"配置"** 面板，配置：
   - **令牌位置**：配置 Token 在请求中的位置（HEADER / JSON_BODY / XML_BODY / FORM_FIELD / MULTIPART_FIELD / URL_PARAM，共 6 种）
   - **令牌方案**：创建一组令牌位置的组合（如"Bearer 认证方案"仅含 Authorization Header），关联方案到用户会话
   - **用户会话**：添加不同权限用户的凭证/Token；支持一键 **"添加匿名用户"**（所有令牌值为空，模拟未登录状态）
   - **判断规则组**：创建规则组（组内条件 AND 组合），设置一个为**活跃规则组**（全局唯一），支持 AND/OR/NOT 运算符
   - **请求范围**：指定需要测试的 URL 模式
2. 开启自动检测，插件会拦截匹配范围的代理流量
3. 自动替换 Token 后重放请求（匿名用户的空 Token 执行"移除"操作），根据活跃规则组判断越权风险
4. 在 **"越权测试"** 面板查看结果（颜色标记：红=疑似越权、绿=安全）

> 判决三层流程：基准无效→错误；活跃规则组命中→越权/安全；无活跃规则组→相似度兜底（≥0.90→越权）。支持从剪贴板解析会话（原始 HTTP / Chrome fetch 格式）。

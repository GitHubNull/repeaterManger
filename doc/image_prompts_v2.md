# Repeater Manager 文生图提示词（补充版 v2）

> 适用模型：通义万相、文心一格、豆包·即梦、混元、可灵、CogView 等国内主流文生图模型
> 与 [image_prompts.md](./image_prompts.md) 互为补充，提供不同视觉切入点
> 设计基础：基于 v2.16.x 实际功能（请求重放、Pool 去重、历史比对、API 规则提取、越权测试、报告生成、ERM 加密存档）

---

## 设计原则

1. **核心视觉符号**
   - **循环箭头**：象征"重放（Repeater）"
   - **盾牌 / 锁**：象征"安全测试 / 加密存档"
   - **多层分身 / 多色身份**：象征"多用户越权测试"
   - **数据池 / 蜂窝**：象征"Pool 去重架构"
   - **棱镜分光**：象征"4 提取源 × 4 提取方法"
   - **卷轴 / 报告页**：象征"PDF/HTML/Markdown 报告"

2. **统一色板**
   - 主色：深空蓝（#0B1A3A）+ 霓虹青（#00E5FF）
   - 强调色：警示红（#FF3B5C，越权告警）+ 安全绿（#00FFA3，安全通过）
   - 点缀色：金色（#FFB800，规则 / 报告）

3. **避免雷同**：本版与 [image_prompts.md](./image_prompts.md) 在切入点上互斥，可任选其一或交叉使用。

## 一、插件 Logo（补充版）

### 提示词 D：棱镜分光（致敬 4×4 提取规则）

```
一个矢量风格的科技图标，主体是一个三角形棱镜，一束白色HTTP请求光线从棱镜左侧射入，从棱镜右侧分散出四束彩色光，分别为青色、绿色、橙色、紫色，象征四种数据提取源。棱镜下方托着一片极简盾牌底座。整体扁平化设计，深蓝渐变背景，光线带轻微辉光，干净利落，无文字，1:1构图，适合作为软件图标和应用商店图标使用。
```
**画幅比例**：1:1
**风格参考**：Vector Icon, Prism Light, Data Extraction Concept

### 提示词 E：蜂窝池去重（致敬 Pool 架构）

```
一个现代极简logo，整体外形为圆角方形容器，内部由多个青色和深蓝色的六边形蜂窝紧密排列，蜂窝中心一个发光的SHA字样小标签。容器外侧环绕一圈细线箭头形成的循环轨道，轨道上有一个亮点正在运动。象征数据池去重与请求循环重放的结合。纯色深背景，扁平矢量风格，无文字，高清矢量icon，缩到16x16依然清晰。
```
**画幅比例**：1:1
**风格参考**：Honeycomb Pool, Minimal Tech Logo

极简风格的科技图标，画面正中是两个并排略微重叠的盾牌剪影，左侧盾牌为蓝色代表"用户A"，右侧盾牌为橙红色代表"用户B"，两个盾牌中间夹着一颗发光的Token钥匙图形，钥匙正在被一道虚线箭头从左盾移动到右盾。盾牌下方一行细小的代码状横线作为底座。整体黑色背景配霓虹色，扁平矢量风格，无文字，1:1正方形构图，强辨识度。

## 二、宣传海报（补充版）

### 提示词 D：指挥官驾驶舱（专业掌控感）

```
一张科幻级别的宣传海报，第一人称视角看向一个未来感的安全测试驾驶舱。正前方是一块巨大弧形主屏，屏幕上分上下两区：上区是Repeater Manager的请求列表，整齐排列彩色标记的HTTP请求条目；下区是历史报文对比视图，左右两栏报文以绿红高亮显示字符级差异。主屏左侧悬浮屏显示越权测试结果矩阵，多个用户身份图标排成一列，对应每行结果以红绿圆点标记。主屏右侧悬浮屏显示SQLite数据库的Pool去重蜂窝可视化与SHA-256哈希流。驾驶舱底部控制台上有金色的API规则书卷和加密锁图标。整体色调深蓝配霓虹青，电影级体积光，超精细渲染，画面顶部预留主标题区，底部预留版本号区。
```
**画幅比例**：3:4 或 2:3（竖版海报）
**风格参考**：Sci-Fi Cockpit, Tactical HUD, Cinematic Tech

---

### 提示词 E：数据池中的灯塔（隐喻去重与持久化）

```
一张概念艺术风格的宣传海报，画面下方是一片由HTTP请求报文文字组成的深蓝色海洋，海面上漂浮着无数发光的六边形数据节点（象征Pool蜂窝去重）。画面正中矗立一座由代码与电路构成的现代灯塔，塔身刻着SQLite的标志和AES-256字样，塔顶发出循环旋转的青色光束扫过整片海洋，每扫过一处就在海面留下亮绿色的"已收录"标记。灯塔脚下有一个水晶质感的盾牌台基。天空是深紫到深蓝渐变的夜空，散布着代码星点。氛围沉稳大气，象征"无论请求多复杂，都能被有序记录与守护"。预留顶部主标题、底部副标题位置。
```
**画幅比例**：2:3（竖版海报）
**风格参考**：Concept Art, Lighthouse Metaphor, Digital Ocean

### 提示词 F：作战地图（流程化展示完整工作流）

```
一张俯视视角的科技作战地图风格海报。地图被分成五个发光的功能区，由发光的数据通路连接成一个闭环：第一区"采集"展示Burp Proxy捕获请求的图标和右键菜单流光；第二区"管理"展示彩色标记的请求列表；第三区"重放"展示请求循环箭头与多次响应历史堆叠；第四区"测试"展示多个用户身份图标与Token替换光线，结果有红绿标记；第五区"报告"展示三本展开的卷轴分别标注PDF、HTML、Markdown。每个功能区下方有一行小字标签。地图整体为深色军绿与霓虹青配色，地形纹理为电路板风格，转折处有金色高亮。画面右上角预留品牌Logo位置，下方预留口号文字区。
```
**画幅比例**：16:9 或 4:3
**风格参考**：Tactical Map, Workflow Infographic, Circuit Topology

## 五、提示词调优技巧

### 5.1 强化关键元素的写法

如果生成结果中某个关键元素不够突出，在提示词末尾追加权重描述：

```
（重点突出：[关键元素名称]，使其位于画面视觉中心，占比不低于30%）
```

### 5.2 中文文字生成的稳定写法

国内模型中通义万相和豆包对中文文字支持较好。如需稳定生成中文文字，建议：

1. 文字内容不超过 8 个汉字
2. 使用"巨大粗体黑色中文文字「XXX」"句式
3. 必要时在负面提示词中加入"文字模糊、错字、繁简混用"

### 5.3 风格一致性

同一项目下的多张图（Logo + 海报 + 封面）建议保持：
- 主色调一致（深蓝 + 霓虹青）
- 强调色一致（警示红 + 安全绿）
- 视觉元素呼应（如都包含循环箭头或盾牌）

### 5.4 通用负面提示词

```
低质量, 模糊, 水印, 文字乱码, 文字错位, 多余手指, 畸形, 杂乱, 过曝, 低分辨率,
素描, 草稿, 黑白, 写实人像, 自然风景, 卡通幼稚, 油画风, 水彩, 像素化,
英文标点错误, 繁体字, 不对称构图（除非要求）
```

## 六、推荐模型与参数

| 模型 | 适合场景 | 推荐分辨率 |
|------|---------|------------|
| 通义万相 v2.1+ | 海报、含中文文字封面 | 1024×1536（海报）/ 1920×1080（封面） |
| 豆包·即梦 3.0 | Logo、概念图 | 1024×1024 |
| 可灵 | 电影感海报、动态封面 | 1280×1920 |
| 混元 | 写实质感海报 | 1024×1536 |
| 文心一格 | 国风变体、插画封面 | 1024×1024 |
| CogView 3 / 4 | 概念性较强的封面 | 1024×1024 |

**通用参数建议**：

- 采样步数：30-40
- 引导强度（CFG）：7-8
- 高清修复：开启（海报必开）
- 种子：满意后固定，便于微调
